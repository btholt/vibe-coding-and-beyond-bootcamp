Beyond Vibe Coding

Welcome! Hello! This is Beyond Vibe Coding, and I am so excited about this course, because it's for a group of people I've wanted to teach for years: people who aren't developers and don't particularly want to become developers — but who want to build real things anyway, and build them well.

Here's the deal. AI can write code now — you've probably heard, possibly at every meeting you've attended since 2023. What the hype leaves out is that there's an enormous difference between generating code and shipping something you understand, can fix, and can grow. That difference is this course. You will not become a professional typist-of-code here. You will become something arguably more useful in this new era: a capable reader of code, a sharp director of the AI that writes it, and a person who can take an idea from blank page to a real, live URL — with a database, user accounts, and your name on the repo — and actually understand what you built. By the end, you'll have shipped several apps and one capstone project that is entirely yours.

We start in the friendliest possible place with my co-instructor Brandon and the fine folks at Replit, and then I'll take the wheel: we'll set up a professional-grade toolchain with VS Code and GitHub Copilot, learn to read the languages of the web, and then go deep — servers, databases, auth, deployment, debugging — the whole path to shipping. Everything at reading depth, everything in service of building.

Who is this course for?

You! No coding experience is required — genuinely none, this course assumes nothing. If you're a PM, a designer, a marketer, an analyst, an accountant, a lawyer, a founder, or just a person with an idea and a healthy suspicion that "the AI will handle it" isn't the whole story, this course was built for you. And if you do have some coding experience, you'll still get a lot here — the back half of the course is about working with AI agents the way the best builders do right now, and that's new territory for almost everyone.

All you need is a computer, curiosity, and a healthy tolerance for my jokes.

Where to file issues

I write these courses with a lot of care, but across this many hours of material, mistakes are inevitable — in the grammar, in the code, occasionally in my opinions (rarely — ask anyone). If you find one, the best thing you can do is open an issue or a pull request on this course's GitHub repo: that way it gets fixed, and everyone who comes after you gets a better course and can Google the same answer you got. I'm always happy to chat on social media, but I can't be tech support for everyone — GitHub is the way. (And conveniently, filing a good issue is literally a skill this course teaches, so consider it extra credit.)
